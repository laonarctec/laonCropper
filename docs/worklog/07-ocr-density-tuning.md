# 07. OCR Density 클러스터링 & 튜닝

## 3가지 변형 비교
사용자 요청으로 Basic / Density(DBSCAN) / Heatmap 3가지 비교.

### Density (DBSCAN)
- 텍스트 중심점을 DBSCAN 클러스터링 → 최대 클러스터의 bbox
- eps = 대각선의 10%, min_samples = 3
- **1번 포스터 문제 해결** (82% → 60%) — 포스터 텍스트 분리 성공
- 6번 카카오톡은 56%만 잡힘 (잘림 ❌)

### Heatmap
- 텍스트 bbox 누적 → threshold → 최대 연결 영역
- 전반적으로 **너무 타이트** — 영수증 상하단 잘림
- 탈락

## DensTuned (최종 승자)
**3단계 파이프라인**:
1. **크기 필터**: 면적 > 중앙값 × 5인 bbox 제거 (포스터/간판 큰 글자 배제)
2. **y가중 스케일**: y좌표 × 0.5 (세로 긴 영수증의 줄 간 연결 촉진)
3. **DBSCAN**: eps = scaled_diag × 10%, min_samples = 3

### 결과
| # | 이미지 | DensTuned |
|---|--------|-----------|
| 1 | 포스터 | ✅ 60%×61% (포스터 분리) |
| 2 | 책상 | ✅ 36%×40% |
| 3 | 키보드 | ⚠️ 68%×69% (키보드 텍스트 포함) |
| 4 | 테이블 | ✅ 36%×53% |
| 5 | 이중 종이 | ✅ 34%×37% |
| 6 | 카카오톡 | ✅ 29%×93% (전체 포함) |

### 핵심 발견
- "요즘 당근" bbox: 중앙값의 **104배** → 크기 필터로 자동 제거
- y_scale=0.5: 세로 텍스트 연결 촉진, 좌우 분리는 유지
- 두 문제(1번 포스터, 6번 카카오톡) 동시 해결
